# Inspect data types.

The [`_type()`](/type#type) is a versatile method that can be used to inspect object type. This method is particularly useful for scripts that rely on 
the type to return a certain value or perform a specific task. The [`_type()`](/type#type) method supports a wide range of data types, including 
[`string`](https://www.tradingview.com/pine-script-reference/v5/#op_string), [`integer`](https://www.tradingview.com/pine-script-reference/v5/#op_int), 
[`boolean`](https://www.tradingview.com/pine-script-reference/v5/#op_bool), [`float`](https://www.tradingview.com/pine-script-reference/v5/#op_float), 
[`color`](https://www.tradingview.com/pine-script-reference/v5/#op_color), [`table`](https://www.tradingview.com/pine-script-reference/v5/#op_table), 
[`line`](https://www.tradingview.com/pine-script-reference/v5/#op_line), [`label`](https://www.tradingview.com/pine-script-reference/v5/#op_label), 
[`box`](https://www.tradingview.com/pine-script-reference/v5/#op_box), [`linefill`](https://www.tradingview.com/pine-script-reference/v5/#op_linefill), 
[`array<type>`](https://www.tradingview.com/pine-script-reference/v5/#op_array), and [`matrix<type>`](https://www.tradingview.com/pine-script-reference/v5/#op_matrix). 
By leveraging the [`_type()`](/type#type) method, Pinescript users can create more robust and flexible scripts that can adapt to a wide range of data inputs.

<Note>Please make sure to import the [`tools`](https://www.tradingview.com/script/KuDBLIOE-tools) library in your script, before following along with the examples.</Note>

<CodeGroup>

```js {{ title: 'Method' }}
// Example testing object types 

varStr = "Hello"
varStr._type() // returns "string".

varFlt = 1.25
varFlt._type() // returns "float".

int varInt = 100
varInt._type() // returns "int".

bool varBool = true
varBool._type() // returns "bool".

color varClr = color.red
varClr._type() // returns "color".

table varTbl = table.new(position, col, row)
varTbl._type() // returns "table".

array<string> varArrS = array.new_string(5,na)
varArrS._type() // returns "array<string>".

matrix<float> varMtxF = matrix.new<float>(4,4,0.5)
varMtxF._type() // returns "matrix<float>".
```

